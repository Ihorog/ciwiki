# ПоДія — Події та сценарії

> Роль: ініціює зміни, тримає темп системи.

## Навіщо
- Визначати, **коли** і **що** запускаємо
- Прив’язка до часових вузлів і станів

## Унікальні риси
- Подієві шаблони: запуск, перехід, завершення
- Узгодження з «Настрій» та «Календарем»

## Типи подій
| Тип | Приклад | Теги |
|---|---|---|
| Ініціація | старт місяця | kickoff, план |
| Маркер | зміна стану | checkpoint |
| Результат | реліз | done, release |

## Механіка
- Тригери → Дія → Запис у хроніку
- Сенсові ваги впливу на модулі

## Далі
- Додати шаблони «сімейних подій»
- Синхрон з Галереєю (автослайди)
